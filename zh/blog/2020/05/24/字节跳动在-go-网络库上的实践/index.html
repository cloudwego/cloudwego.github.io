еП¶дЄАжЦєйЭҐпЉМеЄЄиІБзЪД bytesгАБbufioгАБringbuffer з≠Й buffer еЇУпЉМеЭЗе≠ШеЬ® growth йЬАи¶Б copy еОЯжХ∞зїДжХ∞жНЃпЉМдї•еПКеП™иГљжЙ©еЃєжЧ†ж≥ХзЉ©еЃєпЉМеН†зФ®е§ІйЗПеЖЕе≠Шз≠ЙйЧЃйҐШгАВеЫ†ж≠§жИСдїђеЄМжЬЫеЉХеЕ•дЄАзІНжЦ∞зЪД Buffer ељҐеЉПпЉМдЄАдЄЊиІ£еЖ≥дЄКињ∞дЄ§жЦєйЭҐзЪДйЧЃйҐШгАВ

    • иѓїеЖЩеИЖеИЂжУНдљЬе§іе∞ЊжМЗйТИпЉМзЫЄдЇТдЄНеє≤жЙ∞гАВ
  2. йЂШжХИжЙ©зЉ©еЃє
    • жЙ©еЃєйШґжЃµпЉМзЫіжО•еЬ®е∞ЊжМЗйТИеРОжЈїеК†жЦ∞зЪД block еН≥еПѓпЉМжЧ†йЬА copy еОЯжХ∞зїДгАВ

ињЮжО•е§ЪиЈѓе§НзФ®

  2. Shared map

  3. еНПиЃЃеИЖеМЕеТМзЉЦз†Б

    • е¶ВдљХиѓЖеИЂеЃМжХізЪДиѓЈж±ВеУНеЇФжХ∞жНЃеМЕжШѓињЮжО•е§ЪиЈѓе§НзФ®жЦєж°ИеПѓи°МзЪДеЕ≥йФЃпЉМеЫ†ж≠§йЬАи¶БеЉХеЕ•еНПиЃЃгАВ

ZeroCopy

ињЩйЗМжЙАиѓізЪД ZeroCopyпЉМжМЗзЪДжШѓ Linux жЙАжПРдЊЫзЪД ZeroCopy зЪДиГљеКЫгАВдЄКдЄАзЂ†дЄ≠жИСдїђиѓідЇЖдЄЪеК°е±ВзЪДйЫґжЛЈиіЭпЉМиАМдЉЧжЙАеС®зЯ•пЉМељУжИСдїђи∞ГзФ® sendmsg з≥їзїЯи∞ГзФ®еПСеМЕзЪДжЧґеАЩпЉМ еЃЮйЩЕдЄКдїНзДґжШѓдЉЪдЇІзФЯдЄАжђ°жХ∞жНЃзЪДжЛЈиіЭзЪДпЉМеєґдЄФеЬ®е§ІеМЕеЬЇжЩѓдЄЛињЩдЄ™жЛЈиіЭзЪДжґИиАЧйЭЮеЄЄжШОжШЊгАВдї• 100M дЄЇдЊЛпЉМperf еПѓдї•зЬЛеИ∞е¶ВдЄЛзїУжЮЬпЉЪ

ињЩињШдїЕдїЕжШѓжЩЃйАЪ tcp еПСеМЕзЪДеН†зФ®пЉМеЬ®жИСдїђзЪДеЬЇжЩѓдЄЛпЉМе§ІйГ®еИЖжЬНеК°йГљдЉЪжО•еЕ• Service MeshпЉМжЙАдї•еЬ®дЄАжђ°еПСеМЕдЄ≠пЉМдЄАеЕ±дЉЪжЬЙ 3 жђ°жЛЈиіЭпЉЪдЄЪеК°ињЫз®ЛеИ∞еЖЕж†ЄгАБеЖЕж†ЄеИ∞ sidecarгАБsidecar еЖНеИ∞еЖЕж†ЄгАВ ињЩдљњеЊЧжЬЙе§ІеМЕйЬАж±ВзЪДдЄЪеК°пЉМжЛЈиіЭжЙАеѓЉиЗізЪД cpu еН†зФ®дЉЪзЙєеИЂжШОжШЊпЉМе¶ВдЄЛеЫЊпЉЪ

дЄЇдЇЖиІ£еЖ≥ињЩдЄ™йЧЃйҐШпЉМжИСдїђйАЙжЛ©дЇЖдљњзФ® Linux жПРдЊЫзЪД ZeroCopy APIпЉИеЬ® 4.14 дї•еРОжФѓжМБ sendпЉЫ5.4 дї•еРОжФѓжМБ receiveпЉЙгАВдљЖжШѓињЩеЉХеЕ•дЇЖдЄАдЄ™йҐЭе§ЦзЪДеЈ•з®ЛйЧЃйҐШпЉЪZeroCopy send API еТМеОЯеЕИи∞ГзФ®жЦєеЉПдЄНеЕЉеЃєпЉМжЧ†ж≥ХеЊИе•љеЬ∞еЕ±е≠ШгАВ ињЩйЗМзЃАеНХдїЛзїНдЄАдЄЛ ZeroCopy send зЪДеЈ•дљЬжЦєеЉПпЉЪдЄЪеК°ињЫз®Ли∞ГзФ® sendmsg дєЛеРОпЉМsendmsg дЉЪиЃ∞ељХдЄЛ iovec зЪДеЬ∞еЭАеєґзЂЛеН≥ињФеЫЮпЉМињЩжЧґеАЩдЄЪеК°ињЫз®ЛдЄНиГљйЗКжФЊињЩжЃµеЖЕе≠ШпЉМйЬАи¶БйАЪињЗ epoll з≠ЙеЊЕеЖЕж†ЄеЫЮи∞ГдЄАдЄ™дњ°еПЈи°®жШОжЯРжЃµ iovec еЈ≤зїПеПСйАБжИРеКЯдєЛеРОжЙНиГљйЗКжФЊгАВ зФ±дЇОжИСдїђеєґдЄНеЄМжЬЫжЫіжФєдЄЪеК°жЦєзЪДдљњзФ®жЦєж≥ХпЉМйЬАи¶БеѓєдЄКе±ВжПРдЊЫеРМж≠•жФґеПСзЪДжО•еП£пЉМжЙАдї•еЊИйЪЊеЯЇдЇОзО∞жЬЙзЪД API еРМжЧґжПРдЊЫ ZeroCopy еТМйЭЮ ZeroCopy зЪДжКљи±°пЉЫиАМзФ±дЇО ZeroCopy еЬ®е∞ПеМЕеЬЇжЩѓдЄЛжШѓжЬЙжАІиГљжНЯиАЧзЪДпЉМжЙАдї•дєЯдЄНиГље∞ЖињЩдЄ™дљЬдЄЇйїШиЃ§зЪДйАЙй°єгАВ

дЇОжШѓпЉМе≠ЧиКВиЈ≥еК®ж°ЖжЮґзїДеТМе≠ЧиКВиЈ≥еК®еЖЕж†ЄзїДеРИдљЬпЉМзФ±еЖЕж†ЄзїДжПРдЊЫдЇЖеРМж≠•зЪДжО•еП£пЉЪељУи∞ГзФ® sendmsg зЪДжЧґеАЩпЉМеЖЕж†ЄдЉЪзЫСеРђеєґжЛ¶жИ™еЖЕж†ЄеОЯеЕИзїЩдЄЪеК°зЪДеЫЮи∞ГпЉМеєґдЄФеЬ®еЫЮи∞ГеЃМжИРеРОжЙНдЉЪиЃ© sendmsg ињФеЫЮгАВ ињЩдљњеЊЧжИСдїђжЧ†йЬАжЫіжФєеОЯжЬЙж®°еЮЛпЉМеПѓдї•еЊИжЦєдЊњеЬ∞жО•еЕ• ZeroCopy sendгАВеРМжЧґпЉМе≠ЧиКВиЈ≥еК®еЖЕж†ЄзїДињШеЃЮзО∞дЇЖеЯЇдЇО unix domain socket зЪД ZeroCopyпЉМеПѓдї•дљњеЊЧдЄЪеК°ињЫз®ЛдЄО Mesh sidecar дєЛйЧізЪДйАЪдњ°дєЯиЊЊеИ∞йЫґжЛЈиіЭгАВ

еЬ®дљњзФ®дЇЖ ZeroCopy send еРОпЉМperf еПѓдї•зЬЛеИ∞еЖЕж†ЄдЄНеЖНжЬЙ copy зЪДеН†зФ®пЉЪ

дїО cpu еН†зФ®жХ∞еАЉдЄКзЬЛпЉМе§ІеМЕеЬЇжЩѓдЄЛ ZeroCopy иГље§ЯжѓФйЭЮ ZeroCopy иКВзЬБдЄАеНКзЪД cpuгАВ

жИСдїђе∞ЭиѓХдЇЖеЊИе§ЪзІНеКЮж≥ХеОїдЉШеМЦпЉМдљЖжШѓжФґжХИзФЪеЊЃгАВжЬАзїИпЉМжИСдїђеЃЪдљНеЗЇињЩдЄ™еїґињЯеєґйЭЮжШѓзФ±дЇО Netpoll жЬђиЇЂзЪДеЉАйФАеѓЉиЗізЪДпЉМиАМжШѓзФ±дЇО go зЪДи∞ГеЇ¶еѓЉиЗізЪДпЉМжѓФе¶ВиѓіпЉЪ

  1. зФ±дЇОеЬ® Netpoll дЄ≠пЉМSubReactor жЬђиЇЂдєЯжШѓдЄАдЄ™ goroutineпЉМеПЧи∞ГеЇ¶ељ±еУНпЉМдЄНиГљдњЭиѓБ EpollWait еЫЮи∞ГдєЛеРОй©ђдЄКжЙІи°МпЉМжЙАдї•ињЩдЄАеЭЧдЉЪжЬЙеїґињЯпЉЫ
  2. еРМжЧґпЉМзФ±дЇОзФ®жЭ•е§ДзРЖ I/O дЇЛдїґзЪД SubReactor еТМзФ®жЭ•е§ДзРЖињЮжО•зЫСеРђзЪД MainReactor жЬђиЇЂдєЯжШѓ goroutineпЉМжЙАдї•еЃЮйЩЕдЄКеЊИйЪЊдњЭиѓБеЬ®е§Ъж†ЄжГЕеЖµдєЛдЄЛпЉМињЩдЇЫ Reactor иГљеєґи°МжЙІи°МпЉЫ зФЪиЗ≥еЬ®жЬАжЮБзЂѓжГЕеЖµдєЛдЄЛпЉМеПѓиГљињЩдЇЫ Reactor дЉЪжМВеЬ®еРМдЄАдЄ™ P дЄЛпЉМжЬАзїИеПШжИРдЇЖдЄ≤и°МжЙІи°МпЉМжЧ†ж≥ХеЕЕеИЖеИ©зФ®е§Ъж†ЄдЉШеКњпЉЫ
  3. зФ±дЇО EpollWait еЫЮи∞ГдєЛеРОпЉМSubReactor еЖЕжШѓдЄ≤и°Ме§ДзРЖ I/O дЇЛдїґзЪДпЉМеѓЉиЗіжОТеЬ®жЬАеРОзЪДдЇЛдїґеПѓиГљдЉЪжЬЙйХње∞ЊйЧЃйҐШпЉЫ
  4. еЬ®ињЮжО•е§ЪиЈѓе§НзФ®еЬЇжЩѓдЄЛпЉМзФ±дЇОжѓПдЄ™ињЮжО•зїСеЃЪдЇЖдЄАдЄ™ SubReactorпЉМжХЕеїґињЯеЃМеЕ®еПЦеЖ≥дЇОињЩдЄ™ SubReactor зЪДи∞ГеЇ¶пЉМеѓЉиЗіе∞ЦеИЇдЉЪжЫіеК†жШОжШЊгАВ
